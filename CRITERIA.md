# 平场均匀性判据说明

> 适用于 `heidstar_flat`。代码位置：`heidstar_flat/core/metrics.py`，默认阈值见 `heidstar_flat/config.py`。

---

## 1. 一句话本质

显微相机拍到的每一张图，**实际亮度** = **真实样品亮度** × **照明/光路的不均匀分布（flatfield）**。"平场性检测"就是问一个问题：

> 假如样品本身全是白纸，那这台机器拍出来的图，从中心到角落、从左上到右下，亮度的不均匀程度有多大？能不能在校正后保留足够信噪比？

这套工具的工作是：
1. **估出 flatfield**——`H×W` 的浮点图，每个像素是"此位置相对最亮位置接收的光强百分比"。
2. **从这张 flatfield 上读出 7 个数字指标**，每个数字回答一个具体的工程问题（暗角多深？是否中心被挡？对角线有没有倾斜？……）。
3. **每个指标与阈值比较**，**全部通过 → 通道 PASS；任意一项失败 → 通道 FAIL**（7 项 AND）。

不均匀本身在物理上几乎不可避免（cos⁴ 渐晕 / 光源照度梯度 / 滤光片角度 / CMOS 像素响应），关键是程度可不可控、空间形状是否符合预期。

---

## 2. 判据总览

| # | 名称 | 计算公式 | 方向 | 反映的工程问题 |
|---|---|---|---|---|
| 1 ★ | 稳健 Min/Max | `P1 / P99 × 100%` | **≥** 阈值 | 暗角幅度——校正后还有多少动态范围 |
| 2 ★ | CV 均匀性 | `(1 − σ/μ) × 100%` | **≥** 阈值 | 整体分散度——全场亮度方差大不大 |
| 3 | 九区·四角对称 | `min(四角) / max(四角) × 100%` | **≥** 阈值 | 装配倾斜 / 同心度偏移 |
| 4 | 九区·中心最亮 | `中心格 / max(九格) × 100%` | **≥** 阈值 | 中心被遮挡（污物、手指油） |
| 5 | 九区·最暗格 | `min(九格) / max(九格) × 100%` | **≥** 阈值 | ROI 平均后的最深衰减 |
| 6 | 九区·粗糙度 | `(1 − σ_zone/μ_zone) × 100%` | **≥** 阈值 | 低频空间结构均匀性 |
| 7 | 顶端饱和率 | `% 像素 ≥ 0.99` | **≤** 阈值 | 算法过拟合 / 中心平台异常（反作弊） |

★ 标记的是"幅度类"指标，2 个；其余 4 个是"空间结构类"指标（九区 ROI），加 1 个"分布形状类"反作弊指标。

> **判定结果格式**：`FAIL (Min/Max 28.50% < 30.00% ; 四角对称 65.20% < 70.00%)`——只列失败项；PASS 时列全部数值便于追溯。

---

## 3. 每项判据详解

### 判据 #1 ★ 稳健 Min/Max

```
robust_min_max = P1(flatfield_normalized) / P99(flatfield_normalized) × 100%
```

- **本质**：归一化平场上"1% 最暗位置"相对"1% 最亮位置"的强度比例。
- **为什么用 P1/P99 而不是 min/max**：单个污点或坏点会把 `min` 拖到接近 0，把 `max` 推到 1，导致整张图全是"100%/0% = 0%"的悲观值。用 1%/99% 分位数能抗住孤立异常像素。
- **物理含义**：暗角越深，这个比值越小。30% 意味着"四角接收的光约为中心的 30%"，校正时需要把四角放大 ~3.3 倍——同时把噪声也放大 3.3 倍。
- **每通道阈值不同**（详见 §4）：Yellow/Orange 滤光片本身透光窄、暗角天然重，要求 20–25% 即可；Purple 在我们的样机上反而最均匀，要求 50%。

### 判据 #2 ★ CV 均匀性

```
cv_uniformity = (1 − std(flatfield_normalized) / mean(flatfield_normalized)) × 100%
```

#### CV 是什么

CV (Coefficient of Variation, 变异系数) 是统计学通用指标，公式很简单：

```
CV = σ / μ        （标准差 / 均值）
```

- **σ**：一组数字"互相散得多开"
- **μ**：这组数字的平均水平
- **CV**：把"散度"用"平均值"做归一化，得到一个**无量纲**的相对散度指标

完美一致 → CV = 0；越乱 → CV 越大。

#### 通俗版：全班分数比喻

把"全图每个像素的亮度"换成"全班一百个人的考试分数"：

| 情况 | μ (平均分) | σ (差异) | CV |
|---|---|---|---|
| 所有人都考 80 | 80 | 0 | **0%**（完美一致） |
| 有人 75 有人 85 | 80 | ~5 | **6%**（小差异） |
| 有人 60 有人 100 | 80 | ~20 | **25%**（差异大） |
| 有人 20 有人 100 | 60 | ~40 | **67%**（乱七八糟） |

CV 告诉你这帮人的成绩"是不是基本扎堆，还是天差地别"。搬回平场，就是几百万个像素亮度"是不是基本一致（好平场），还是有的很亮有的很暗（差平场）"。

#### 用 4 个像素手算一遍

```
[0.8  0.9]              μ = 0.85
[0.7  1.0]              σ ≈ 0.112
                        CV = 0.112 / 0.85 ≈ 13.2%
```

换成更糟的：

```
[0.3  0.9]              μ = 0.65
[0.4  1.0]              σ ≈ 0.295
                        CV ≈ 45.4%
```

#### 为什么报告里看到的是 "75%" 而不是 "25%"

代码用的是 **`(1 − CV) × 100%`**，把"低 = 好"翻成"高 = 好"，跟其他 6 项判据方向一致——避免 7 项里 6 项 "≥ 阈值"、1 项 "≤ 阈值"导致看报告时混淆。

| 原始 CV | 报告里 "CV 均匀性" | 含义 |
|---|---|---|
| 0 % | **100 %** | 完美一致 |
| 13 % | 87 % | 较好 |
| 25 % | 75 % | 卡在默认阈值上 |
| 45 % | 55 % | 比较差，FAIL |

**默认阈值 75%** = σ/μ ≤ 25% = 标准差不超过平均值的 1/4。工程上"还能用"的稳健下限。

#### 与判据 #1 的互补

| | 判据 #1 Min/Max | 判据 #2 CV |
|---|---|---|
| 关心啥 | **最暗 vs 最亮**（极值） | **整体方差**（所有像素相互的散度） |
| 怎么挂 | 有一片角落特别黑 → 挂 | 整体亮度参差不齐 → 挂 |
| 反例 | 暗角不深但中间起伏明显 → #1 过 / #2 可能挂 | 一片亮一片暗但还算分两片均匀 → #1 挂 / #2 可能过 |

#1 抓"极值有多差"、#2 抓"整体有多乱"——任何一个挂都 FAIL，两种失效模式都被拦。

### 判据 #3–#6 九区 ROI 系列

把归一化平场切成 **3×3 = 9 块**（用 `np.linspace(0, H, 4)` 等分），算每块的均值。9 个均值放在一起，导出 4 个独立指标——这层粗粒度抹掉了像素级噪声，专门看"大尺度形状"。

**九区编号**（行列）：

```
0 1 2     ← 顶部
3 4 5     ← 中部（4 = 中心）
6 7 8     ← 底部
```

四角 = `{0, 2, 6, 8}`，中心 = `{4}`。

#### #3 四角对称性

```
corner_symmetry = min(z0, z2, z6, z8) / max(z0, z2, z6, z8) × 100%
```

- **本质**：四个角的最暗块 vs 最亮块的比值。
- **诊断什么**：如果是装配倾斜或同心度偏移，会出现"左上很亮 + 右下很暗"这种对角不对称；理想光路四角应大致等值。
- **默认 70%**：四角中最暗的至少是最亮的 70%。

#### #4 中心最亮

```
center_to_max = z4 / max(z0..z8) × 100%
```

- **本质**：中心块亮度相对九块中最亮块的比例。
- **诊断什么**：理论上中心块应该是最亮的（cos⁴ 渐晕规律 + 光源中心最强）。如果中心被遮挡（污物、镜头脏、机械干涉），中心块会从"最亮"掉下来，这一项就 < 100%。
- **默认 95%**：允许 5% 的浮动；低于 95% 通常意味着中心确有异常。

#### #5 最暗格

```
min_zone_to_max = min(z0..z8) / max(z0..z8) × 100%
```

- **本质**：九块中最暗块相对最亮块的比例，与 #1 类似但在低频尺度。
- **与 #1 的区别**：#1 是像素级 P1/P99，会受少数像素影响；#5 是已经做过区域平均（ROI ~ H/3 × W/3）的最暗块，关心的是"是不是有一整块明显偏暗"。
- **默认 40%**：可以理解为"中等粗粒度看，最暗的 1/9 至少有最亮 1/9 的 40% 亮度"。

#### #6 九格粗糙度

```
nine_zone_uniformity = (1 − std(z0..z8) / mean(z0..z8)) × 100%
```

- **本质**：把 9 个块均值当一组样本，算 1 − CV。
- **与 #2 的区别**：#2 是全像素 CV，会被高频噪声拉低；#6 只看 9 个块均值的 CV，专门看大尺度均匀性。
- **默认 75%**。

> **#3–#6 为什么要拆成 4 个**：每一项指向一种**具体的物理失效模式**，FAIL 时直接告诉操作员"是哪种问题"。例如：
> - 只 #4 挂 → 中心被挡
> - 只 #3 挂 → 装配倾斜
> - 只 #5 挂 → 局部缺陷
> - #3–#6 全挂 → 整体光路有大问题

### 判据 #7 顶端饱和率（反作弊）

```
top_saturation = (#{pixels | flatfield_normalized ≥ 0.99}) / total_pixels × 100%
```

- **本质**：归一化后强度 ≥ 0.99 的像素占比。**方向反转：低 = 好**，越低越正常，超过阈值 FAIL。
- **为什么需要这一项**：平场估计算法（特别是 BaSiC 的低秩+稀疏分解）在某些病态输入下会"过拟合"——估出来的平场是一个超大的中心平台（很多像素都贴近 1.0），周围掉到很低。这种平场的前 6 项指标会**意外漂亮**（极值幅度大、CV 高、九区对称），但实际上是数学伪影，对真实图像校正起反作用。
- **怎么识别**：正常的平场是渐变的，归一化强度 = 0.99 的像素应该只是顶部小峰。一旦出现"中心一大片像素全压在 0.99 以上"，这个百分比会从 < 1% 跳到 10%+。
- **默认 5%**：归一化强度 ≥ 0.99 的像素占比 ≤ 5%。

> **设计动机**：在 BaSiC 算法路径下踩过实际坑——某些曝光偏高的通道会被估出"中心 95% 的像素都是 1.0"的伪平场，前 6 项全 PASS 但人眼一看就是错的。第 7 项就是专门拦截这种情况。

## 5. 算法关键细节

- **归一化**：所有指标在 `flatfield_normalized = flatfield / max(flatfield)` 上算，确保 max ≡ 1。
- **稳健化**：`P1 / P99` 抗孤立坏点（污染像素、热点）。
- **位置记录**：同时记录原始 `min` / `max` 的像素 `(row, col)`，UI 热力图会标这两个点——肉眼立刻分辨"暗角型" vs "中心异常型"。
- **九区均分**：用 `np.linspace(0, H, 4)` / `np.linspace(0, W, 4)`，所以**奇数 H/W 时各块大小可能差 1 像素**，这是有意为之的简化，对均值影响 < 1‰。

---

## 6. 阈值调整工作流

1. **先跑一组已知合格的数据**，记下 7 项实际数值。
2. **每项阈值留 ~10–15 个百分点的余量**（实测 - 余量 = 阈值）。例如实测 Yellow #1 = 33%，阈值定 20–22% 比较稳。
3. **GUI 「通道偏好与设置」**里调整，会持久化到：
   - Linux/macOS: `~/.config/heidstar_flat/config.json`
   - Windows: `%APPDATA%\heidstar_flat\config.json`
4. **#7 顶端饱和率**：除非你确认平场估计算法换了新版本/参数，否则**不要随手调高**——这是反作弊用的，调高就废了它的作用。

---

## 7. 失效模式速查

| FAIL 项 | 优先排查 |
|---|---|
| 仅 #1 | 暗角太重——检查光源功率、滤光片对中、物镜清洁度 |
| 仅 #4 | 中心被挡——检查物镜前端、Bayer 滤光片、样品台是否有杂物 |
| 仅 #3 | 倾斜/同心度——检查 LED 阵列对中、模组安装垂直度 |
| 仅 #5 | 局部缺陷——某个角落被遮挡 / 一块镜片有划痕 |
| #3 #4 #5 #6 同挂 | 整体光路严重偏 / 选错通道 |
| #1–#6 全 PASS 但 #7 挂 | 平场估计算法对该通道过拟合——换一组瓦片或检查曝光是否过曝 |
| 全 PASS 但人眼觉得"中心一大块全白" | 应该 #7 拦截，没拦住说明 `top_saturation_threshold` 调得太松，回 5% |

# Part B：杂散光评估（独立于平场）

> 与平场评估**互不相干**：自己的数据采集（关激发暗场图）、自己的 PASS/FAIL 判定（2 项 AND）、自己的 PDF 报告。代码位置：`heidstar_flat/core/stray_light.py`，默认阈值见 `heidstar_flat/config.py`。

---

## B.1 一句话本质

显微相机理论上"没光时输出 0"，**实际上不会**。即使把所有激发光关掉、用黑布盖住样品台，传感器仍然会读出非零值——这些"凭空冒出来的亮度"统称**本底光（dark level）**，物理来源主要是：

- **暗电流**：CMOS 像素本身的热噪声（曝光时间越长越严重）
- **偏置**：读出电路的电压偏置
- **环境光漏入**：壳体缝隙、镜筒密封不严、相邻光源漏光
- **通道间串扰**：激发滤光片不全密封，另一波段光漏进当前通道

平场评估在意"照射场的不均匀"，杂散光评估在意"**没有任何照射时**还能收到多少光，以及这些杂光是不是均匀分布"。

> **怎么测**：关闭所有激发 LED / 激光器 → 用与正常扫描相同的曝光/增益拍 N 张瓦片 → 这就是该通道的"暗场图"。本评估从这些暗场图算出 2 个数字指标。

---

## B.2 判据总览

| # | 名称 | 计算公式 | 方向 | 反映的工程问题 |
|---|---|---|---|---|
| **DC1** ★ | 本底强度 | `dark_mean / sensor_max × 100%` | **≤** 阈值 | 整体本底有多亮——是否存在普遍性杂光泄漏 |
| **DC2** ★ | 本底空间均匀性 | `(1 − std(zone)/mean(zone)) × 100%` | **≥** 阈值 | 暗本底是否空间一致——某一片明显亮意味着局部杂光斑 |
| **DC3** ★ | DSNU 像素级 | `std(mean_image) / sensor_max × 100%` | **≤** 阈值 | 像素间固定模式噪声（EMVA 1288 DSNU 粗版本） |
| **DC4** ★ | 时间噪声底 | `median(std_over_time) / sensor_max × 100%` | **≤** 阈值 | 无信号下的随机噪声底，决定 SNR 分母 |
| **DC5** ★ | 热像素密度 | `(#{mean_pixel > μ + Kσ}) / total × 100%` | **≤** 阈值 | 孤立异常亮点个数（DC1-DC4 都看不到这些） |

**5 项 AND**：全过 → PASS；任一项挂 → FAIL。

`sensor_max` 按数据 dtype 推断：uint8 = 255，uint16 = 65535，float = 1.0。
DC5 的 K=10（保守，只抓极端异常）；μ 是 mean_image 的均值、σ 是 mean_image 的空间 std（即 DC3 的绝对值）。

---

## B.3 每项判据详解

### 判据 DC1：本底强度

```
DC1 = dark_mean / sensor_max × 100%
```

#### 本质

把"暗场图的平均灰度"除以"传感器满量程"，得到一个百分比。这个百分比就是"传感器在没光照时'凭空收到'的能量比例"。

#### 通俗解释

**类比**：关灯睡觉。理想情况下房间应该是纯黑（0% 亮度），但实际上：

| 情况 | "本底亮度" 类比 |
|---|---|
| 关灯 + 拉窗帘 + 戴眼罩 | 接近 0%，几乎纯黑 |
| 关灯 + 窗外有路灯 | 1%，有一点点能看到东西 |
| 关灯 + 房间里有显示器待机灯 | 5%，能勉强看清家具轮廓 |
| 没关灯（白天） | 80%，本来就不该是这个场景 |

对显微相机来说，"关灯 + 拉窗帘"对应**关激发 + 黑布盖样品台**。理想 DC1 接近 0，实际总有几个百分点是正常的（暗电流 + 偏置永远存在）。

#### 数字示例

假设是 uint16 sCMOS（sensor_max = 65535）：

| 暗场 mean | DC1 | 意味着 |
|---|---|---|
| 100 ADU | **0.15%** | 优秀，典型 sCMOS 在短曝光下的本底 |
| 600 ADU | **0.92%** | 卡在默认阈值 1% 上 |
| 1500 ADU | **2.3%** | 超阈值 FAIL，可能曝光过长 + 漏光 |
| 30000 ADU | **45.8%** | 严重异常，几乎肯定没关激发或盖子没盖 |

#### 默认阈值 1% 的含义

`DC1 ≤ 1%` 等价于：暗场平均灰度不超过传感器满量程的 1/100。对 uint16 来说，mean ≤ 655 ADU；对 uint8 来说，mean ≤ 2.55 ADU。

这是个**宽松的工程阈值**，工业级 sCMOS 在合理曝光下应该轻松通过。如果连这个都过不了，说明肯定有问题（曝光过长 / 增益过高 / 真的漏光）。

#### 物理失效来源

- **DC1 偏高 + DC2 正常**：暗电流积累过多 → 曝光时间太长，或机器温度偏高
- **DC1 偏高 + DC2 也低**：有杂光漏入且不均匀 → 壳体密封问题 / 通道间串扰

---

### 判据 DC2：本底空间均匀性

```
DC2 = (1 − std(zone) / mean(zone)) × 100%
   其中 zone[0..8] 是 9 区 ROI 均值
```

#### 本质

把暗场均值图切成 3×3 = 9 块，算每块的均值，然后看这 9 个数字相互之间的**散度**（用 CV 思路，σ/μ）。再用 `1 − CV` 翻成"高 = 好"。

完美均匀的暗场：9 块均值一模一样 → CV = 0 → DC2 = 100%

#### 通俗解释

**类比**：关灯后摸黑找东西，看房间各处是不是"黑得一样深"：

| 情况 | DC2 类比 |
|---|---|
| 全屋一致漆黑 | 100%（理想） |
| 厨房和客厅差不多黑，卧室稍亮一点 | 85% |
| 一半房间漆黑，另一半被外面路灯照得能看见家具 | 50% |
| 一束阳光从天窗射进客厅 | 20% |

对显微相机来说，**"哪一片明显比其他地方亮"** = 那一片有杂光斑漏入。常见原因：

- 镜筒接缝处漏光
- 二向色镜没装正、有反射
- 相邻通道的激发光从滤光片边缘绕过来
- 样品台没盖紧

#### 数字示例

假设暗场 9 区均值如下（uint16，理想全场都该接近 ~80 ADU）：

```
情况 A（均匀）           情况 B（角落漏光）       情况 C（中间有杂光斑）
80  82  79               80  82  79               80  82  79
81  80  82               81  80  82               81  450 82
80  81  79               80  81  220              80  81  79
```

- **A**: mean=80, std=1.0, CV=1.3% → DC2 = **98.7%**  ✅
- **B**: mean=96, std=43.4, CV=45.2% → DC2 = **54.8%**  ✗ FAIL（一角异常）
- **C**: mean=121, std=121.7, CV=100% → DC2 = **0%**   ✗ FAIL（中间巨大杂光斑）

#### 默认阈值 60% 的含义

`DC2 ≥ 60%` 等价于：9 块均值的 σ/μ ≤ 40%。这是个**中等严格的工程阈值**——理想暗场应该 95%+，60% 给定了相当大的容忍空间。

> 如果你装备的 sCMOS 暗电流本身就有可见的空间梯度（某些工业相机的特性），默认 60% 可能过严，根据真实数据调整。

#### 物理失效来源（与 DC1 的互补）

| DC1 | DC2 | 含义 |
|---|---|---|
| ✅ 低 | ✅ 高 | 暗场低且均匀——理想状态 |
| ❌ 高 | ✅ 高 | 整体本底偏高但全场一致 → 暗电流积累 / 曝光太长，不一定有杂光 |
| ✅ 低 | ❌ 低 | 总体很暗但**某一片明显亮** → 局部杂光斑（最值得查的失效模式） |
| ❌ 高 | ❌ 低 | 又高又乱 → 严重问题，多重原因叠加 |

**单看 DC1 不够**：可能整体平均很低但局部一片亮（藏在平均里看不出来），这时 DC1 通过 DC2 拦截。
**单看 DC2 也不够**：可能空间均匀但本底整体偏高（暗电流积累），这时 DC2 通过 DC1 拦截。

---

### 判据 DC3：DSNU 像素级（Dark Signal Non-Uniformity）

```
DC3 = std(mean_image) / sensor_max × 100%
   其中 mean_image = stack.mean(axis=0)  即每像素跨 N 帧时间平均
```

#### 本质

DC2 看的是 **9 个大块**之间的差异（粗粒度，~1000×1000 像素 ROI）；
DC3 看的是 **每个单独像素**之间的差异（细粒度，1×1 像素）。

物理上对应**固定模式噪声 (Fixed-Pattern Noise, FPN) 的暗成分**：CMOS 制造工艺让每个像素的暗电流和偏置有微小差异，这种差异不会随时间变化（只随温度漂移），所以叫"固定"。

EMVA 1288 标准里的 DSNU 严格定义是单位电子数；我们这里用归一化到 sensor_max 的百分比，便于跨相机比较。

#### 通俗解释

DC2 像在 9 块区域里量"哪片暗得不一样"；DC3 像在每个像素位置量"这个像素是不是有自己的固定偏差"。

**类比**：一张桌面是不是平的：
- DC2：把桌面分 9 块，看哪块凹哪块凸（看大坑）
- DC3：用手指摸过每个点，看每个点的高低差（看微观毛刺）

理想完美传感器 DC3 = 0；实际工业 CMOS 1-5 ADU，sCMOS < 2 ADU，CCD 甚至 < 1 ADU。

#### 数字示例（uint16）

| 相机类型 | 典型 DSNU 绝对值 | DC3 % | 评价 |
|---|---|---|---|
| 高端 sCMOS（Andor Zyla / Hamamatsu Flash） | 1-2 ADU | 0.002-0.003% | 优秀 |
| 中端工业 CMOS | 5-10 ADU | 0.008-0.015% | 良好 |
| 入门工业 CMOS | 20-50 ADU | 0.03-0.08% | 一般 |
| 故障 / 老化 | 300+ ADU | 0.5%+ | 不可接受 |

#### 默认阈值 0.5% 的含义

`DC3 ≤ 0.5%` 等价于 DSNU ≤ 327 ADU on uint16。这是**非常宽松**的工程下限——任何健康的工业相机都能通过；卡到这条线说明硬件已经显著退化。

如果你的相机是优质 sCMOS，建议把这条线收紧到 0.05% 或更低，否则它失去拦截作用。

#### 与 DC1/DC2 的区别

| 故障类型 | DC1 | DC2 | DC3 |
|---|---|---|---|
| 普遍漏光 | ✗ | ✓ | ✓ |
| 局部杂光斑（一片亮） | 可能✓ | ✗ | 可能 ✗ |
| 大面积一致的高暗电流 | ✗ | ✓ | ✓ |
| **大量像素各自有小偏差** | ✓ | ✓ | ✗ |
| 个别热像素 | ✓ | ✓ | 可能 ✓ |

DC3 专抓"像素级 fixed-pattern noise"，前两者抓不到。

---

### 判据 DC4：时间噪声底（Temporal Noise Floor）

```
DC4 = median(pixel_std_over_time) / sensor_max × 100%
   其中 pixel_std_over_time[r,c] = std(stack[:, r, c])  即每像素跨 N 帧时间 std
```

#### 本质

DC3 看的是"像素之间的固定差异"；DC4 看的是"**同一个像素**跨多帧的随机抖动"。两者**完全独立**：

- DC3 是空间噪声（同一时刻像素与像素的差）
- DC4 是时间噪声（同一像素不同时刻的差）

物理上 DC4 对应**读出噪声 + 暗电流散粒噪声**。是相机 SNR 计算的分母——再亮的信号也得至少超过这个噪声底才能被检测。

EMVA 1288 把这个量叫 **σ_temporal**。

#### 通俗解释

**类比**：看一只表的秒针是不是稳定。
- DC3 像把 100 只表摆一起，看哪只指针位置不一样（**空间差**）
- DC4 像盯一只表看一分钟，看它指针有没有抖动（**时间抖**）

DC4 抓的是相机 **"凭空冒出来的随机噪声"**。理想 sCMOS read noise < 2 e-，对应几 ADU。

#### 数字示例（uint16）

| 相机类型 | 典型时间噪声 | DC4 % | 评价 |
|---|---|---|---|
| 高端 sCMOS read noise 1 e- | ~1 ADU | 0.0015% | 优秀 |
| 中端 sCMOS read noise 2-3 e- | 2-3 ADU | 0.003-0.005% | 良好 |
| 工业 CMOS read noise 5-10 e- | 5-10 ADU | 0.008-0.015% | 一般 |
| 故障 / 接地不良 / 干扰 | 100+ ADU | 0.15%+ | 异常 |

#### 默认阈值 1% 的含义

`DC4 ≤ 1%` 等价于时间噪声 ≤ 655 ADU on uint16。**极度宽松**——正常相机都在 0.005-0.05% 之间，1% 只能拦截"线路接地不良 / 严重电磁干扰"这种硬故障。

正常工况下建议收紧到 0.05% 或按相机规格调。

#### 注意事项

DC4 至少需要 **N ≥ 2 帧**才有意义；只有 1 帧时算不出时间 std，本工具自动退化为 0（永远 PASS）。建议杂散光采集时至少拍 5 张以上瓦片。

---

### 判据 DC5：热像素密度

```
DC5 = #{pixels | mean_pixel > μ + K·σ} / total_pixels × 100%
   其中 μ = mean(mean_image), σ = std(mean_image) (即 DSNU 绝对值), K = 10
```

#### 本质

DC1-DC4 都是**统计量**（均值、std），对孤立异常像素不敏感（被平均/中位数稀释掉）。
DC5 专门**计数**：明显高于本底 K 倍 σ 的像素有多少个。

物理上对应**热像素 / 死像素 / 缺陷像素**：CMOS 工艺缺陷产生的少数像素，暗电流是正常像素的几十倍。每个相机出厂时 hot pixel 个数都有 spec。

#### 通俗解释

**类比**：一张全黑照片里能数出几个"鬼影亮点"。
- DC1-DC4 像问"照片整体亮度多少 / 均匀不均匀 / 噪声大不大"
- DC5 像问"照片上一共有几颗特别亮的孤立光点"

K=10 的意思是：只算"亮度高于本底 10 倍 DSNU 的像素"——非常保守，普通暗角 / 噪声波动不会被误判，只有真正的故障像素才会被计入。

#### 数字示例

3000×2656 sensor 共 7,968,000 像素：

| 情况 | 热像素个数 | DC5 % | ppm |
|---|---|---|---|
| 全新 sCMOS（< 10 个热像素） | 10 | 0.0001% | 1.3 ppm |
| 典型工业 CMOS spec ≤ 100 ppm | 800 | 0.01% | 100 ppm |
| 严重老化 / 工艺故障 | 8000+ | 0.1%+ | 1000 ppm |

#### 默认阈值 0.01%（≈ 100 ppm）的含义

`DC5 ≤ 0.01%` 对应工业级 CMOS 常见 spec **100 ppm 热像素**上限。优质 sCMOS 实际只有 1-10 ppm，远远低于这条线。

如果你的相机厂家给出更严格的 spec（比如 10 ppm），可以把阈值收紧到 0.001%。

#### 与其他 DC 的差异

| 故障类型 | DC1 | DC2 | DC3 | DC4 | DC5 |
|---|---|---|---|---|---|
| 100 个分散热像素 | 通过（被平均） | 通过 | 几乎通过 | 通过 | **✗** |
| 1 片连成块的热区 | 可能✗ | ✗ | ✗ | 通过 | 可能✗ |
| 单个特别亮的死像素 | 通过 | 通过 | 通过 | 通过 | **✗** |

DC5 的不可替代性：**抓孤立的、空间分散的、统计平均看不出来的少数异常像素**。

## B.4 默认阈值

| 项 | 默认 | 配置字段 |
|---|---|---|
| DC1 本底强度 | ≤ 1.0 % | `stray_dc_threshold` |
| DC2 本底均匀性 | ≥ 60 % | `stray_zone_dc_uniformity_threshold` |
| DC3 DSNU 像素级 | ≤ 0.5 % | `stray_dsnu_threshold` |
| DC4 时间噪声底 | ≤ 1.0 % | `stray_temporal_noise_threshold` |
| DC5 热像素密度 | ≤ 0.01 % (≈ 100 ppm) | `stray_hot_pixel_threshold` |

阈值全通道共用（不像平场的 #1 是按通道差异化设置）。理由：暗本底/DSNU/读出噪声/热像素这些主要由相机和机壳决定，与具体荧光通道关系不大。

**这些默认值是宽松起点**——对工业级 CMOS 都能轻松通过，主要拦截硬故障。对高端 sCMOS 建议按相机 spec 至少收紧 1-2 个数量级（参见各 DC 节"默认阈值含义"段落给的相机分级对照表）。

---

## B.5 数据采集要求

| 项 | 要求 |
|---|---|
| 激发光 | **全部关闭**（LED / 激光器 / 其他光源） |
| 环境 | 关闭室内主光源（至少避免直射机器） |
| 样品 | 黑布或盖子盖住样品台（防止反射光） |
| 曝光 / 增益 | **与正常扫描相同**——这样测的是实际工作条件下的本底，不是理论极限 |
| 瓦片数 | 建议 ≥ 5；多张瓦片平均能抑制单帧噪声，让 9 区分析更稳定 |
| 文件格式 | 与正常扫描一致（`*_<Color>/Images/IMG*.tif`），程序按相同结构发现通道 |

> **为什么"曝光与正常扫描一致"很重要**：DC1 与曝光时间线性相关（暗电流积累）。如果你用 1 ms 测 DC1=0.1%，但实际工作用 100 ms，那真实工况下 DC1 ≈ 10%——测试和实际脱节。

---

## B.6 算法关键细节

- **sensor_max 由 dtype 推断**：uint8=255、uint16=65535、float=1.0；自定义 dtype 走 `np.iinfo(dtype).max`。
- **9 区均值取法**：先对所有瓦片在像素维度做平均得到 H×W 的"代表暗场图"，再切 3×3 ROI 算各块均值。先平均后切区**抗单帧随机噪声**。
- **最亮像素位置标注**：从代表暗场图取 `argmax`，作为"疑似杂光斑"位置在 UI/PDF 上标红圈，方便人工排查。
- **不做归一化到 [0,1]**：DC1 直接除以 `sensor_max` 后表达为百分比；DC2 在原始灰度量级上算 CV，单位无关。

## B.7 失效模式速查

| 状况 | 优先排查 |
|---|---|
| DC1 ✗ + DC2 ✓ | 整体本底偏高但均匀——降低曝光 / 增益；检查机器温度 |
| DC1 ✓ + DC2 ✗ | 有**局部杂光斑**——重点查"最亮像素位置"标红那一片：镜筒接缝、二向色镜对中、邻通道光泄漏 |
| DC1 ✗ + DC2 ✗ | 又高又乱——多重问题；先盖严样品台、关大灯重测，仍 FAIL 则查光机内部 |
| 仅 DC3 ✗（DC1/DC2 都过） | 像素间固定偏差大——传感器制造缺陷 / 工艺退化 / 多次老化测试；通常无法现场修复 |
| 仅 DC4 ✗ | 时间噪声异常高——电源/接地干扰、附近电磁源、传感器读出电路问题；检查线缆屏蔽 |
| 仅 DC5 ✗ | 热像素超规格——传感器局部缺陷 / 长期高温老化；少量可在校正阶段 mask 掉 |
| DC3 + DC4 + DC5 同挂 | 相机本身已严重退化或故障，建议换机或返厂 |
| 不同通道 DC2 差异巨大 | 通道间串扰——某通道的激发光漏到其他通道的光路 |
| 全 PASS 但**真实样品**扫描底色偏高 | 你测的暗场与真实扫描条件可能不一致（曝光 / 增益 / 环境光不同） |

---

## B.8 不做的（边界声明）

- **鬼影 / 反射伴生峰检测**：需要点光源样品（单颗荧光珠 / 针孔）才能算径向响应曲线；暗场图里没有亮点，无法反推。将来若有此类采集再追加单独模块。
- **通道间串扰矩阵**：需要"轮流只开一个通道激发，其他通道也拍图"的特定采集协议；当前不要求此种数据。
- **遮幕眩光 (Veiling glare) / 视场外漏光**：需要 ISO 18844 风格的黑白棋盘格采集；超出当前测试范围。

如果以后需要这些维度，可以在 `core/` 下新增独立模块（不混入 `stray_light.py`），UI 上新增独立面板（不混入「杂散光检测」Tab）。
