# 平场均匀性判据说明

> 适用于 `heidstar_flat`。代码位置：`heidstar_flat/core/metrics.py`，默认阈值见 `heidstar_flat/config.py`。

---

## 1. 一句话本质

显微相机拍到的每一张图，**实际亮度** = **真实样品亮度** × **照明/光路的不均匀分布（flatfield）**。"平场性检测"就是问一个问题：

> 假如样品本身全是白纸，那这台机器拍出来的图，从中心到角落、从左上到右下，亮度的不均匀程度有多大？能不能在校正后保留足够信噪比？

这套工具的工作是：
1. **估出 flatfield**——`H×W` 的浮点图，每个像素是"此位置相对最亮位置接收的光强百分比"。
2. **从这张 flatfield 上读出 7 个数字指标**，每个数字回答一个具体的工程问题（暗角多深？是否中心被挡？对角线有没有倾斜？……）。
3. **每个指标与阈值比较**，**全部通过 → 通道 PASS；任意一项失败 → 通道 FAIL**（7 项 AND）。

不均匀本身在物理上几乎不可避免（cos⁴ 渐晕 / 光源照度梯度 / 滤光片角度 / CMOS 像素响应），关键是程度可不可控、空间形状是否符合预期。

---

## 2. 判据总览

| # | 名称 | 计算公式 | 方向 | 反映的工程问题 |
|---|---|---|---|---|
| 1 ★ | 稳健 Min/Max | `P1 / P99 × 100%` | **≥** 阈值 | 暗角幅度——校正后还有多少动态范围 |
| 2 ★ | CV 均匀性 | `(1 − σ/μ) × 100%` | **≥** 阈值 | 整体分散度——全场亮度方差大不大 |
| 3 | 九区·四角对称 | `min(四角) / max(四角) × 100%` | **≥** 阈值 | 装配倾斜 / 同心度偏移 |
| 4 | 九区·中心最亮 | `中心格 / max(九格) × 100%` | **≥** 阈值 | 中心被遮挡（污物、手指油） |
| 5 | 九区·最暗格 | `min(九格) / max(九格) × 100%` | **≥** 阈值 | ROI 平均后的最深衰减 |
| 6 | 九区·粗糙度 | `(1 − σ_zone/μ_zone) × 100%` | **≥** 阈值 | 低频空间结构均匀性 |
| 7 | 顶端饱和率 | `% 像素 ≥ 0.99` | **≤** 阈值 | 算法过拟合 / 中心平台异常（反作弊） |

★ 标记的是"幅度类"指标，2 个；其余 4 个是"空间结构类"指标（九区 ROI），加 1 个"分布形状类"反作弊指标。

> **判定结果格式**：`FAIL (Min/Max 28.50% < 30.00% ; 四角对称 65.20% < 70.00%)`——只列失败项；PASS 时列全部数值便于追溯。

---

## 3. 每项判据详解

### 判据 #1 ★ 稳健 Min/Max

```
robust_min_max = P1(flatfield_normalized) / P99(flatfield_normalized) × 100%
```

- **本质**：归一化平场上"1% 最暗位置"相对"1% 最亮位置"的强度比例。
- **为什么用 P1/P99 而不是 min/max**：单个污点或坏点会把 `min` 拖到接近 0，把 `max` 推到 1，导致整张图全是"100%/0% = 0%"的悲观值。用 1%/99% 分位数能抗住孤立异常像素。
- **物理含义**：暗角越深，这个比值越小。30% 意味着"四角接收的光约为中心的 30%"，校正时需要把四角放大 ~3.3 倍——同时把噪声也放大 3.3 倍。
- **每通道阈值不同**（详见 §4）：Yellow/Orange 滤光片本身透光窄、暗角天然重，要求 20–25% 即可；Purple 在我们的样机上反而最均匀，要求 50%。

### 判据 #2 ★ CV 均匀性

```
cv_uniformity = (1 − std(flatfield_normalized) / mean(flatfield_normalized)) × 100%
```

#### CV 是什么

CV (Coefficient of Variation, 变异系数) 是统计学通用指标，公式很简单：

```
CV = σ / μ        （标准差 / 均值）
```

- **σ**：一组数字"互相散得多开"
- **μ**：这组数字的平均水平
- **CV**：把"散度"用"平均值"做归一化，得到一个**无量纲**的相对散度指标

完美一致 → CV = 0；越乱 → CV 越大。

#### 通俗版：全班分数比喻

把"全图每个像素的亮度"换成"全班一百个人的考试分数"：

| 情况 | μ (平均分) | σ (差异) | CV |
|---|---|---|---|
| 所有人都考 80 | 80 | 0 | **0%**（完美一致） |
| 有人 75 有人 85 | 80 | ~5 | **6%**（小差异） |
| 有人 60 有人 100 | 80 | ~20 | **25%**（差异大） |
| 有人 20 有人 100 | 60 | ~40 | **67%**（乱七八糟） |

CV 告诉你这帮人的成绩"是不是基本扎堆，还是天差地别"。搬回平场，就是几百万个像素亮度"是不是基本一致（好平场），还是有的很亮有的很暗（差平场）"。

#### 用 4 个像素手算一遍

```
[0.8  0.9]              μ = 0.85
[0.7  1.0]              σ ≈ 0.112
                        CV = 0.112 / 0.85 ≈ 13.2%
```

换成更糟的：

```
[0.3  0.9]              μ = 0.65
[0.4  1.0]              σ ≈ 0.295
                        CV ≈ 45.4%
```

#### 为什么报告里看到的是 "75%" 而不是 "25%"

代码用的是 **`(1 − CV) × 100%`**，把"低 = 好"翻成"高 = 好"，跟其他 6 项判据方向一致——避免 7 项里 6 项 "≥ 阈值"、1 项 "≤ 阈值"导致看报告时混淆。

| 原始 CV | 报告里 "CV 均匀性" | 含义 |
|---|---|---|
| 0 % | **100 %** | 完美一致 |
| 13 % | 87 % | 较好 |
| 25 % | 75 % | 卡在默认阈值上 |
| 45 % | 55 % | 比较差，FAIL |

**默认阈值 75%** = σ/μ ≤ 25% = 标准差不超过平均值的 1/4。工程上"还能用"的稳健下限。

#### 与判据 #1 的互补

| | 判据 #1 Min/Max | 判据 #2 CV |
|---|---|---|
| 关心啥 | **最暗 vs 最亮**（极值） | **整体方差**（所有像素相互的散度） |
| 怎么挂 | 有一片角落特别黑 → 挂 | 整体亮度参差不齐 → 挂 |
| 反例 | 暗角不深但中间起伏明显 → #1 过 / #2 可能挂 | 一片亮一片暗但还算分两片均匀 → #1 挂 / #2 可能过 |

#1 抓"极值有多差"、#2 抓"整体有多乱"——任何一个挂都 FAIL，两种失效模式都被拦。

### 判据 #3–#6 九区 ROI 系列

把归一化平场切成 **3×3 = 9 块**（用 `np.linspace(0, H, 4)` 等分），算每块的均值。9 个均值放在一起，导出 4 个独立指标——这层粗粒度抹掉了像素级噪声，专门看"大尺度形状"。

**九区编号**（行列）：

```
0 1 2     ← 顶部
3 4 5     ← 中部（4 = 中心）
6 7 8     ← 底部
```

四角 = `{0, 2, 6, 8}`，中心 = `{4}`。

#### #3 四角对称性

```
corner_symmetry = min(z0, z2, z6, z8) / max(z0, z2, z6, z8) × 100%
```

- **本质**：四个角的最暗块 vs 最亮块的比值。
- **诊断什么**：如果是装配倾斜或同心度偏移，会出现"左上很亮 + 右下很暗"这种对角不对称；理想光路四角应大致等值。
- **默认 70%**：四角中最暗的至少是最亮的 70%。

#### #4 中心最亮

```
center_to_max = z4 / max(z0..z8) × 100%
```

- **本质**：中心块亮度相对九块中最亮块的比例。
- **诊断什么**：理论上中心块应该是最亮的（cos⁴ 渐晕规律 + 光源中心最强）。如果中心被遮挡（污物、镜头脏、机械干涉），中心块会从"最亮"掉下来，这一项就 < 100%。
- **默认 95%**：允许 5% 的浮动；低于 95% 通常意味着中心确有异常。

#### #5 最暗格

```
min_zone_to_max = min(z0..z8) / max(z0..z8) × 100%
```

- **本质**：九块中最暗块相对最亮块的比例，与 #1 类似但在低频尺度。
- **与 #1 的区别**：#1 是像素级 P1/P99，会受少数像素影响；#5 是已经做过区域平均（ROI ~ H/3 × W/3）的最暗块，关心的是"是不是有一整块明显偏暗"。
- **默认 40%**：可以理解为"中等粗粒度看，最暗的 1/9 至少有最亮 1/9 的 40% 亮度"。

#### #6 九格粗糙度

```
nine_zone_uniformity = (1 − std(z0..z8) / mean(z0..z8)) × 100%
```

- **本质**：把 9 个块均值当一组样本，算 1 − CV。
- **与 #2 的区别**：#2 是全像素 CV，会被高频噪声拉低；#6 只看 9 个块均值的 CV，专门看大尺度均匀性。
- **默认 75%**。

> **#3–#6 为什么要拆成 4 个**：每一项指向一种**具体的物理失效模式**，FAIL 时直接告诉操作员"是哪种问题"。例如：
> - 只 #4 挂 → 中心被挡
> - 只 #3 挂 → 装配倾斜
> - 只 #5 挂 → 局部缺陷
> - #3–#6 全挂 → 整体光路有大问题

### 判据 #7 顶端饱和率（反作弊）

```
top_saturation = (#{pixels | flatfield_normalized ≥ 0.99}) / total_pixels × 100%
```

- **本质**：归一化后强度 ≥ 0.99 的像素占比。**方向反转：低 = 好**，越低越正常，超过阈值 FAIL。
- **为什么需要这一项**：平场估计算法（特别是 BaSiC 的低秩+稀疏分解）在某些病态输入下会"过拟合"——估出来的平场是一个超大的中心平台（很多像素都贴近 1.0），周围掉到很低。这种平场的前 6 项指标会**意外漂亮**（极值幅度大、CV 高、九区对称），但实际上是数学伪影，对真实图像校正起反作用。
- **怎么识别**：正常的平场是渐变的，归一化强度 = 0.99 的像素应该只是顶部小峰。一旦出现"中心一大片像素全压在 0.99 以上"，这个百分比会从 < 1% 跳到 10%+。
- **默认 5%**：归一化强度 ≥ 0.99 的像素占比 ≤ 5%。

> **设计动机**：在 BaSiC 算法路径下踩过实际坑——某些曝光偏高的通道会被估出"中心 95% 的像素都是 1.0"的伪平场，前 6 项全 PASS 但人眼一看就是错的。第 7 项就是专门拦截这种情况。

## 5. 算法关键细节

- **归一化**：所有指标在 `flatfield_normalized = flatfield / max(flatfield)` 上算，确保 max ≡ 1。
- **稳健化**：`P1 / P99` 抗孤立坏点（污染像素、热点）。
- **位置记录**：同时记录原始 `min` / `max` 的像素 `(row, col)`，UI 热力图会标这两个点——肉眼立刻分辨"暗角型" vs "中心异常型"。
- **九区均分**：用 `np.linspace(0, H, 4)` / `np.linspace(0, W, 4)`，所以**奇数 H/W 时各块大小可能差 1 像素**，这是有意为之的简化，对均值影响 < 1‰。

---

## 6. 阈值调整工作流

1. **先跑一组已知合格的数据**，记下 7 项实际数值。
2. **每项阈值留 ~10–15 个百分点的余量**（实测 - 余量 = 阈值）。例如实测 Yellow #1 = 33%，阈值定 20–22% 比较稳。
3. **GUI 「通道偏好与设置」**里调整，会持久化到：
   - Linux/macOS: `~/.config/heidstar_flat/config.json`
   - Windows: `%APPDATA%\heidstar_flat\config.json`
4. **#7 顶端饱和率**：除非你确认平场估计算法换了新版本/参数，否则**不要随手调高**——这是反作弊用的，调高就废了它的作用。

## 7. 失效模式速查

| FAIL 项 | 优先排查 |
|---|---|
| 仅 #1 | 暗角太重——检查光源功率、滤光片对中、物镜清洁度 |
| 仅 #4 | 中心被挡——检查物镜前端、Bayer 滤光片、样品台是否有杂物 |
| 仅 #3 | 倾斜/同心度——检查 LED 阵列对中、模组安装垂直度 |
| 仅 #5 | 局部缺陷——某个角落被遮挡 / 一块镜片有划痕 |
| #3 #4 #5 #6 同挂 | 整体光路严重偏 / 选错通道 |
| #1–#6 全 PASS 但 #7 挂 | 平场估计算法对该通道过拟合——换一组瓦片或检查曝光是否过曝 |
| 全 PASS 但人眼觉得"中心一大块全白" | 应该 #7 拦截，没拦住说明 `top_saturation_threshold` 调得太松，回 5% |
